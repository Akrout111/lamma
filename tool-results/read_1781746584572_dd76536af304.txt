    31→    31→## 1. نظرة عامة على المشروع
    32→    32→
    33→    33→### التعريف المختصر
    34→    34→
    35→    35→**لَمَّة (Lamma)** هي منصة كويتية لتجمعات حميمية مُنتقاة — تتعامل مع الفعاليات كحوارٍ، لا كمعاملة. تستلهم المشروع من ثقافة **الديوانية الكويتية** (التي أضافتها UNESCO لقائمة التراث الثقافي غير المادي عام 2025) لكن بصورة عصرية شبابية.
    36→    36→
    37→    37→### لماذا "لَمَّة"؟
    38→    38→
    39→    39→- **لمة** في اللهجة الكويتية = تجمّع صغير دافئ
    40→    40→- مرادف حداثي للديوانية لكن أكثر شبابية وعصرية
    41→    41→- سهلة النطق بالعربية والإنجليزية (Lamma)
    42→    42→- تحمل إحساس "اللمّ الشمل" — التجمع حول فكرة
    43→    43→
    44→    44→### الفكرة في جملة واحدة
    45→    45→
    46→    46→> منصة لتجمعات حميمية (15-80 شخص) مُنتقاة بعناية في الكويت، حيث يتقدّم المهتمون بطلب حضور بدل شراء تذكرة، ويتم اختيارهم بناءً على مدى ملاءمتهم لموضوع التجمع، مع تجربة تحريرية ومجتمعية قبل وبعد كل لمة.
    47→    47→
    48→    48→### النوع
    49→    49→
    50→    50→منصة **C2C Marketplace** للتجمّعات الحميمية — بين منصات الفعاليات التقليدية وشبكات التواصل المجتمعي.
    51→    51→
    52→    52→### المكدس التقني
    53→    53→
    54→    54→| الطبقة | التقنية |
    55→    55→|--------|---------|
    56→    56→| Frontend | Next.js 16 (App Router) + React 19 + TypeScript |
    57→    57→| Styling | Tailwind CSS 4 + Radix UI |
    58→    58→| Database | PostgreSQL (Supabase) + pgvector |
    59→    59→| ORM | Prisma 6.19+ |
    60→    60→| Runtime | Bun |
    61→    61→| Containerization | Docker multi-stage |
    62→    62→| Hosting | Railway |
    63→    63→| Realtime | Supabase Realtime |
    64→    64→| Payments | Tap Payments (KNET) |
    65→    65→| AI | OpenAI embeddings + pgvector |
    66→    66→| Maps | Mapbox |
    67→    67→| External APIs | AlAdhan (مواقيت الصلاة) |
    68→    68→| CMS | Sanity أو محتوى مدمج في Prisma |

    72→    72→## 2. الفلسفة والتموضع
    73→    73→
    74→    74→### الفلسفة الأساسية
    75→    75→
    76→    76→```
    77→    77→الموقع الأول:  تصفّح  →  اشترِ  →  احضر
    78→    78→لَمَّة:         اكتشف  →  تقدّم بطلب  →  تجمّع  →  شارك
    79→    79→```
    80→    80→
    81→    81→### المبادئ الأربعة
    82→    82→
    83→    83→#### 1. الحميمية على الحجم
    84→    84→- لا فعاليات بـ 500+ شخص
    85→    85→- كل لمة بين 15-80 شخص كحد أقصى
    86→    86→- الحضور بناءً على **الطلب والموافقة**، وليس الشراء المباشر
    87→    87→
    88→    88→#### 2. التحرير على المعاملات
    89→    89→- كل لمة لها **قصة** تُروى قبل وبعد
    90→    90→- مجلة "Letters from a gathering" — رسائل تحريرية طويلة
    91→    91→- تصنيف حسب **المواضيع** (Topics) وليس الفئات (Categories)
    92→    92→
    93→    93→#### 3. المجتمع على الجمهور
    94→    94→- الحضور المتراكم يبني **سمعة**
    95→    95→- مطابقة ذكية بين الحضور بنفس الاهتمامات
    96→    96→- العضوية لها **مستويات** (Newcomer → Regular → Curator)
    97→    97→
    98→    98→#### 4. الوعي الثقافي على العمومية
    99→    99→- جدولة تحترم **مواقيت الصلاة**
   100→   100→- فلترة حسب **الصيغة** (رجال/نساء/عائلات/مختلط)
   101→   101→- تقويم يدعم **الهجري** بجانب الميلادي
   102→   102→- محتوى **ثنائي اللغة** بشكل أصيل (ليس ترجمة آلية)

   146→   146→## 4. الجمهور المستهدف
   147→   147→
   148→   148→### الشخصيات الأساسية (User Personas)
   149→   149→
   150→   150→#### الشخصية 1: نورة (28 سنة، كويتية)
   151→   151→- **الوظيفة:** مصمّمة جرافيك
   152→   152→- **الدخل:** 1500 KWD/شهر
   153→   153→- **السلوك:** تبحث عن تجارب **أصيلة وحميمية**، تكره الحفلات المزدحمة
   154→   154→- **الألم:** الفعاليات في الكويت إما超大 مزدحمة أو مكررة
   155→   155→- **ما تريده من لَمَّة:** لقاء أشخاص يشاركونها اهتماماتها بشكل أعمق
   156→   156→
   157→   157→#### الشخصية 2: أحمد (34 سنة، كويتي)
   158→   158→- **الوظيفة:** مهندس برمجيات
   159→   159→- **الدخل:** 2500 KWD/شهر
   160→   160→- **السلوك:** يحضر ورش عمل، يريد **تعلّم** وليس فقط ترفيه
   161→   161→- **الألم:** الورش باهظة (300+ KWD) وغير منتقاة
   162→   162→- **ما يريده من لَمَّة:** ماستر كلاسات بأسعار معقولة مع أناس جادين
   163→   163→
   164→   164→#### الشخصية 3: سارة (24 سنة، مغتربة مصرية)
   165→   165→- **الوظيفة:** مديرة تسويق
   166→   166→- **الدخل:** 900 KWD/شهر
   167→   167→- **السلوك:** جديدة في الكويت، تريد **بناء شبكة علاقات**
   168→   168→- **الألم:** صعوبة دخول المجتمع الكويتي المغلق
   169→   169→- **ما تريده من لَمَّة:** تجمعات ترحيبية متنوعة ثقافياً
   170→   170→
   171→   171→#### الشخصية 4: خالد (42 سنة، كويتي)
   172→   172→- **الوظيفة:** **صاحب تجمّع** — فنان تشكيلي
   173→   173→- **الألم:** يريد تنظيم ورش فنية لكن لا يوجد منصة تناسب حجمه الصغير
   174→   174→- **ما يريده من لَمَّة:** أدوات احترافية لإدارة ورشه بدون تعقيد Eventbrite
   175→   175→
   176→   176→### المقاييس الديموغرافية المستهدفة
   177→   177→
   178→   178→- **العمر:** 22-45 سنة (تركيز على 25-35)
   179→   179→- **الجنسية:** 60% كويتيون، 30% مغتربون عرب، 10% مغتربون غير عرب
   180→   180→- **الجنس:** متوازن (مع احترام الصيغ العائلية/المنفصلة)
   181→   181→- **الدخل:** متوسط وعالي (500+ KWD/شهر)
   182→   182→- **اللغة:** عربي وإنجليزي بطلاقة

   462→   462→
   463→   463→هذه هي الميزات الـ 10 التي تميّز لَمَّة عن أي منصة كويتية أخرى:
   464→   464→
   465→   465→### 8.1 ⭐ الحضور بناءً على الطلب (Application-based Attendance)
   466→   466→
   467→   467→**كيف يعمل:**
   468→   468→1. يتصفّح المستخدم اللمات
   469→   469→2. يضغط "تقديم طلب الحضور"
   470→   470→3. يملأ نموذج: motivation, background, dietary needs
   471→   471→4. صاحب التجمع يراجع الطلبات مع AI match scores
   472→   472→5. يتم القبول/الرفض/قائمة الانتظار
   473→   473→
   474→   474→**لماذا مميّز:** يخلق حصرية وجودة — مثل Sofar Sounds. كل لمة = أشخاص اختاروا أن يكونوا هناك فعلاً.
   475→   475→
   476→   476→### 8.2 🕌 الجدولة الواعية لمواقيت الصلاة
   477→   477→
   478→   478→**كيف يعمل:**
   479→   479→```typescript
   480→   480→// تكامل مع AlAdhan API
   481→   481→const prayerTimes = await fetch(
   482→   482→  `https://api.aladhan.com/v1/timingsByCity?city=Kuwait&method=4`
   483→   483→);
   484→   484→
   485→   485→// عند إنشاء لمة، يظهر تحذير إذا تعارضت مع صلاة
   486→   486→if (gatheringStart <= maghribTime && gatheringEnd >= maghribTime) {
   487→   487→  showWarning("وقتك يتزامن مع صلاة المغبر — هل تريد تعديله؟");
   488→   488→}
   489→   489→
   490→   490→// عرض شارة على صفحة اللمة
   491→   491→<PrayerTimeBadge 
   492→   492→  startTime={gathering.start}
   493→   493→  endTime={gathering.end}
   494→   494→  prayerTimes={prayerTimes}
   495→   495→/>
   496→   496→```
   497→   497→
   498→   498→**لماذا مميّز:** لا توجد منصة في الكويت تفعل هذا. احترام عميق للثقافة.
   499→   499→
   500→   500→### 8.3 📝 التصنيف التحريري حسب الموضوع (Topic Taxonomy)
   501→   501→
   502→   502→بدل التصنيف التقليدي (موسيقى/رياضة/مؤتمرات)، استخدم تصنيفاً تحريرياً:

   505→   505→- "ذاكرة المكان"        (لمات في أماكن تاريخية)

   507→   507→- "الضوء والظل"         (تصوير فني)
   508→   508→- "حكايات الطعام"        (تجارب طعام)
   509→   509→- "أصوات جديدة"          (موسيقى ناشئة)
   510→   510→- "ما بعد العمل"          (networking للمحترفين)
   511→   511→- "روح الكتاب"            (نوادي كتاب)
   512→   512→- "حركات الجسد"           (يوجا، رقص، حركة)

   514→   514→
   515→   515→كل موضوع له **صفحة** (مثل category page) لكن بأسلوب مجلة.
   516→   516→

   530→   530→```

   532→   532→```
   533→   533→
   534→   534→- **Chat مباشر** للحاضرين فقط
   535→   535→- **صور لحظية** يرفعها الحضور
   536→   536→- **لقطاتMoments** — إشارات قصيرة ("هذه اللحظة رائعة!")
   537→   537→- **خريطة الموقع** للمتأخرين
   538→   538→- **عدّاد الوقت المتبقي**

   544→   544→
   545→   545→بعد كل لمة، يُنشر مقال طويل (2000-3000 كلمة) يحكي:
   546→   546→- ما حدث في اللمة
   547→   547→- اقتباسات من الحاضرين
   548→   548→- صور
   549→   549→- تأملات
   550→   550→- اقتراحات لمات قادمة
   551→   551→
   552→   552→هذا يبني **مجلة رقمية** — محتوى SEO قوي + سبب للعودة.
   553→   553→
   554→   554→### 8.7 🧠 مطابقة الحضور بالـ AI (Attendee Matching)
   555→   555→
   556→   556→**كيف يعمل:**
   557→   557→1. كل مستخدم يملأ **اهتماماته** عند التسجيل
   558→   558→2. نحسب **embeddings** لكل مستخدم باستخدام OpenAI
   559→   559→3. نخزّنها في pgvector
   560→   560→4. عند التقديم على لمة، نحسب الـ cosine similarity مع الحاضرين المعتمدين الآخرين
   561→   561→5. نعرض **match score** لصاحب التجمع: "هذا الشخص يتطابق 87% مع الحاضرين الآخرين"
   562→   562→
   563→   563→```typescript
   564→   564→// مثال على الحساب
   565→   565→const userEmbedding = await openai.embeddings.create({
   566→   566→  model: 'text-embedding-3-small',
   567→   567→  input: user.interests.join(' '),
   568→   568→});
   569→   569→
   570→   570→await prisma.$executeRaw`
   571→   571→  UPDATE "User" 
   572→   572→  SET embedding = ${userEmbedding.data[0].embedding}::vector 
   573→   573→  WHERE id = ${user.id}
   574→   574→`;
   575→   575→
   576→   576→// عند التقديم
   577→   577→const matchScore = await prisma.$queryRaw`
   578→   578→  SELECT 1 - (embedding <=> ${userEmbedding}::vector) as similarity
   579→   579→  FROM "User" 
   580→   580→  WHERE id IN (${approvedAttendeeIds})
   581→   581→  LIMIT 1
   582→   582→`;
   583→   583→```
   584→   584→
   585→   585→### 8.8 🎯 فلترة حسب الجنس/الصيغة (Format Filters)
   586→   586→
   587→   587→كل لمة لها **صيغة**:
   588→   588→- `MEN_ONLY` — رجال فقط
   589→   589→- `WOMEN_ONLY` — نساء فقط
   590→   590→- `FAMILY` — عائلات (مع أطفال)

   592→   592→
   593→   593→الفلترة واضحة في الـ UI، مع شرح لكل صيغة.
   594→   594→
   595→   595→### 8.9 📋 قائمة انتظار مرتّبة بالاستحقاق (Merit-ranked Waitlist)
   596→   596→
   597→   597→عند امتلاء اللمة:
   598→   598→- المستخدم يُضاف لقائمة الانتظار

   601→   601→  - مستوى العضوية (Curator > Regular > Newcomer)
   602→   602→  - مدى ملاءمة الاهتمامات
   603→   603→- عند الانسحاب، يُتاح المكان لأعلى شخص في القائمة
   604→   604→
   605→   605→### 8.10 🎛️ لوحة تحكم صاحب التجمع (Host Concierge Dashboard)
   606→   606→
   607→   607→أدوات احترافية تشمل:
   608→   608→- إنشاء لمة بنموذج **متعدد الخطوات** (مع حفظ تلقائي)

   613→   613→- **Concierge AI**: اقتراحات لتحسين وصف اللمة

  1209→  1209→// src/lib/prayer-times.ts
  1210→  1210→const ALADHAN_BASE = 'https://api.aladhan.com/v1';
  1211→  1211→

  1237→  1237→      maghrib: new Date(data.data.timings.Maghrib),
  1238→  1238→      isha: new Date(data.data.timings.Isha),
  1239→  1239→    },

  1245→  1245→  end: Date,
  1246→  1246→  prayerTimes: PrayerTime
  1247→  1247→): { conflict: boolean; prayer?: string } {
  1248→  1248→  const prayers = ['fajr', 'dhuhr', 'asr', 'maghrib', 'isha'];
  1249→  1249→

  1253→  1253→

  1263→  1263→### 11.2 OpenAI Embeddings (AI Matching)
  1264→  1264→
  1265→  1265→```typescript
  1266→  1266→// src/lib/embeddings.ts
  1267→  1267→import OpenAI from 'openai';
  1268→  1268→
  1269→  1269→const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });
  1270→  1270→
  1271→  1271→export async function generateUserEmbedding(user: User): Promise<number[]> {
  1272→  1272→  const text = [
  1273→  1273→    user.name.ar,
  1274→  1274→    user.name.en,
  1275→  1275→    user.bio?.ar,
  1276→  1276→    user.bio?.en,
  1277→  1277→    ...user.interests,
  1278→  1278→  ].filter(Boolean).join(' ');
  1279→  1279→  
  1280→  1280→  const response = await openai.embeddings.create({
  1281→  1281→    model: 'text-embedding-3-small',
  1282→  1282→    input: text,
  1283→  1283→  });
  1284→  1284→  
  1285→  1285→  return response.data[0].embedding;
  1286→  1286→}
  1287→  1287→
  1288→  1288→export async function calculateMatchScore(
  1289→  1289→  userId: string,
  1290→  1290→  gatheringId: string
  1291→  1291→): Promise<number> {
  1292→  1292→  // احصل على embeddings للمستخدم
  1293→  1293→  const userResult = await prisma.$queryRaw`
  1294→  1294→    SELECT embedding FROM "User" WHERE id = ${userId}
  1295→  1295→  ` as { embedding: number[] }[];
  1296→  1296→  
  1297→  1297→  // احصل على embeddings للحاضرين المعتمدين الآخرين
  1298→  1298→  const attendeesResult = await prisma.$queryRaw`
  1299→  1299→    SELECT u.embedding 
  1300→  1300→    FROM "User" u
  1301→  1301→    JOIN "Application" a ON a.userId = u.id
  1302→  1302→    WHERE a.gatheringId = ${gatheringId}
  1303→  1303→      AND a.status = 'APPROVED'
  1304→  1304→      AND u.id != ${userId}
  1305→  1305→      AND u.embedding IS NOT NULL
  1306→  1306→  ` as { embedding: number[] }[];
  1307→  1307→  
  1308→  1308→  if (attendeesResult.length === 0) return 50; // default
  1309→  1309→  
  1310→  1310→  // احسب متوسط الـ cosine similarity
  1311→  1311→  const similarities = attendeesResult.map(attendee => {
  1312→  1312→    return cosineSimilarity(userResult[0].embedding, attendee.embedding);
  1313→  1313→  });
  1314→  1314→  
  1315→  1315→  const avgSimilarity = similarities.reduce((a, b) => a + b, 0) / similarities.length;
  1316→  1316→  return Math.round(avgSimilarity * 100);
  1317→  1317→}
  1318→  1318→
  1319→  1319→function cosineSimilarity(a: number[], b: number[]): number {
  1320→  1320→  let dot = 0, normA = 0, normB = 0;
  1321→  1321→  for (let i = 0; i < a.length; i++) {
  1322→  1322→    dot += a[i] * b[i];
  1323→  1323→    normA += a[i] * a[i];
  1324→  1324→    normB += b[i] * b[i];
  1325→  1325→  }
  1326→  1326→  return dot / (Math.sqrt(normA) * Math.sqrt(normB));
  1327→  1327→}
  1328→  1328→```

  1682→  1682→## 14. نشر المشروع (Railway + Supabase)
  1683→  1683→
  1684→  1684→### 14.1 إعداد Supabase
  1685→  1685→
  1686→  1686→1. أنشئ مشروع Supabase جديد باسم `lamma`
  1687→  1687→2. كلمة المرور بدون `$` (مثل `Lamma2026Secure`)
  1688→  1688→3. المنطقة: EU West 1
  1689→  1689→4. فعّل pgvector extension:
  1690→  1690→   ```sql
  1691→  1691→   CREATE EXTENSION IF NOT EXISTS vector;
  1692→  1692→   CREATE EXTENSION IF NOT EXISTS pg_trgm;
  1693→  1693→   ```
  1694→  1694→5. انسخ Connection URLs:
  1695→  1695→   - `DATABASE_URL` = Transaction pooler (port 6543) + `?pgbouncer=true`
  1696→  1696→   - `DIRECT_URL` = Direct connection (port 5432)
  1697→  1697→
  1698→  1698→### 14.2 إعداد Railway
