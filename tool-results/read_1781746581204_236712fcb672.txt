    61→## 2. منهجية Vibe Coding 7.0
    62→
    63→### 2.1 ما هي منهجية Vibe Coding 7.0؟
    64→
    65→هي منهجية تطوير برمجي تعتمد على ثلاث ركائز أساسية تضمن جودة الكود وأمانه وقابلية صيانته عند بناء المشاريع باستخدام وكلاء الذكاء الاصطناعي. هذه المنهجية طُوّرت عبر 13 سبنت (Sprint) من التطوير المتواصل، وقد أثبتت فعاليتها في تقليل الأخطاء الحرجة بنسبة 60% بعد تطبيقها بشكل صحيح.
    66→
    67→### 2.2 الركيزة الأولى: مبدأ المصدر الواحد (Single Source Principle)
    68→
    69→**المفهوم:** يجب أن يكون هناك وكيل ذكاء اصطناعي واحد فقط هو "المهندس المعماري" للمشروع، يجمع كل القرارات المعمارية والقواعد في ملف واحد قابل للوصول من جميع الوكلاء.
    70→
    71→**التطبيق العملي:**
    72→
    73→- أنشئ ملف `ARCHITECTURE.md` في جذر المشروع يحتوي على كل القواعد.
    74→- كل وكيل فرعي (subagent) يجب أن يقرأ هذا الملف قبل البدء بأي مهمة.
    75→- لا تسمح لوكلاء متعددين باتخاذ قرارات معمارية متضاربة.
    76→- أي قرار جديد يضاف لملف `ARCHITECTURE.md` ويُوثّق بتاريخه وسببه.
    77→
    78→**قالب `ARCHITECTURE.md`:**
    79→
    80→```markdown
    81→# ARCHITECTURE.md — قرارات معمارية للمشروع
    82→
    83→## Tech Stack
    84→- Next.js 16 (App Router)
    85→- TypeScript (strict mode)
    86→- Clerk for Authentication
    87→- Prisma + PostgreSQL
    88→- Tailwind CSS v4
    89→- shadcn/ui components
    90→
    91→## Rules
    92→- Rule 1: Prisma singleton pattern (lib/prisma.ts)
    93→- Rule 2: Clerk auth pattern (auth() + clerkMiddleware)
    94→- Rule 3: API response format (successResponse/errorResponse)
    95→- Rule 4: Decimal → .toString() before JSON
    96→- Rule 5: kebab-case file names
    97→- Rule 6: CSS Logical Properties (RTL-first)
    98→- Rule 7: shadcn/ui only (no Material UI/Ant Design)
    99→- Rule 8: useTranslations() for all UI text
   100→- Rule 9: Zod v4 with `message` for every field
   101→- Rule 10: catch blocks use `unknown` (not `any`)
   102→- Rule 11: Role protection on server side
   103→- Rule 12: No `any` type (use `unknown` or explicit interfaces)
   104→- Rule 13: No Server Actions — use API Routes
   105→- Rule 14: Link from `@/i18n/routing`
   106→- Rule 15: `@/` import alias only
   107→
   108→## Decisions Log
   109→- 2026-06-15: اخترنا Clerk بدلاً من NextAuth لتوفير أسابيع من العمل
   110→- 2026-06-16: استخدمنا Decimal(10, 3) للعملة الكويتية (KWD = 1000 فلس)
   111→- 2026-06-17: اعتمدنا SQLite للتطوير المحلي و PostgreSQL للإنتاج
   112→```
   113→
   114→### 2.3 الركيزة الثانية: حلقة Micro-Sprint (Micro-Sprint Loop)
   115→
   116→**المفهوم:** بدلاً من العمل بسبنتات طويلة (أسبوعين أو أكثر)، قسّم العمل إلى سبنتات صغيرة جداً (30 دقيقة إلى 3 ساعات لكل سبنت) مع دورة تطوير سريعة.
   117→
   118→**هيكل السبنت الواحد:**
   119→
   120→```
   121→1. تحديد الهدف الواحد للسبنت (Goal)
   122→2. كتابة الكود (Code)
   123→3. الاختبار الفوري (Test)
   124→4. المراجعة الذاتية (Self-Review)
   125→5. التحقق من القواعد (Rules Check)
   126→6. الدمج (Merge)
   127→```
   128→
   129→**الفوائد المثبتة:**
   130→
   131→- اكتشاف الأخطاء في وقت مبكر جداً.
   132→- تقليل التراجع عن القرارات (rework) بنسبة 70%.
   133→- الحفاظ على تركيز الوكيل على مهمة واحدة.
   134→- إمكانية العودة لنسخة سابقة بسهولة عند الفشل.
   135→
   136→### 2.4 الركيزة الثالثة: Stop Gate QA (بوابة التوقف لضمان الجودة)
   137→
   138→**المفهوم:** في نهاية كل سبنت، يجب التوقف عند "بوابة جودة" لا يمكن تجاوزها إلا بعد التحقق من معايير محددة (راجع قسم 12 في هذا الملف).
   139→
   140→> ⚠️ **تحذير حرج:** لا تتجاوز بوابة الجودة أبداً. إذا فشل أي بند، أصلحه قبل المضي قدماً. تجاوز البوابة هو السبب الأول لتراكم الديون التقنية.
   141→
   142→### 2.5 دورة حياة المشروع الكاملة
   143→
   144→```
   145→[التخطيط] → [الإعداد] → [السبنت 1] → [Stop Gate] → [السبنت 2] → [Stop Gate] →
   146→   ... → [التدقيق النهائي] → [الإطلاق] → [المراقبة]
   147→```
   148→
   149→كل مرحلة لها مدخلات ومخرجات محددة، ولا يمكن الانتقال للمرحلة التالية إلا بإكمال مخرجات المرحلة السابقة.

   153→## 3. المكدس التقني القياسي (Tech Stack)
   154→
   155→### 3.1 الأساسيات (غير قابلة للتغيير)
   156→
   157→| الطبقة | التقنية | السبب |
   158→|--------|---------|-------|
   159→| **Framework** | Next.js 16 (App Router) | أحدث إصدار، SSR/SSG، RSC، API Routes |
   160→| **Language** | TypeScript 5 (strict) | أمان الأنواع، كشف الأخطاء قبل التشغيل |
   161→| **Styling** | Tailwind CSS 4 | سرعة، اتساق، responsive، دعم RTL |
   162→| **UI Components** | shadcn/ui (New York) | جودة عالية، قابلية تخصيص، لا قفل بائع |
   163→| **Icons** | Lucide React | خفيف، tree-shaking |
   164→| **Database ORM** | Prisma 6.x | TypeScript-first + migrations آمنة |
   165→| **Database (Dev)** | SQLite | تطوير سريع، لا حاجة لخادم |
   166→| **Database (Prod)** | PostgreSQL (Supabase) | موثوقية، JSONB، Enums، pooling |
   167→| **Auth** | Clerk (مُفضّل) أو NextAuth.js v4 | جاهز، آمن، يوفّر أسابيع |
   168→| **i18n** | next-intl | RTL/LTR، تعدد اللغات، تكامل مع App Router |
   169→| **State (Client)** | Zustand | بساطة وأداء |
   170→| **State (Server)** | TanStack Query | caching، optimistic updates |
   171→| **Validation** | Zod v4 | تحقق من الأنواع في الـ runtime |
   172→| **Animation** | Framer Motion | حركات ناعمة |
   173→| **3D (اختياري)** | Three.js + React Three Fiber | تأثيرات بصرية متقدمة |
   174→| **Package Manager** | Bun | سرعة فائقة + دعم TypeScript أصلي |
   175→| **Runtime** | Bun | أداء أفضل من Node.js |
   176→| **Containerization** | Docker (multi-stage) | بيئة متّسقة بين dev و production |
   177→| **Base Image** | `oven/bun:1-slim` | صورة رسمية خفيفة |
   178→| **Hosting** | Railway | نشر سهل + healthcheck + variables |
   179→| **Database Hosting** | Supabase | طبقة مجانية + PgBouncer pooling |
   180→
   181→### 3.2 البنية التحتية
   182→
   183→```
   184→Next.js App (port 3000)
   185→    │
   186→    ├── Caddy Gateway (port 81 → exposed externally)
   187→    │       └── XTransformPort query → mini-services
   188→    │
   189→    ├── Mini-Services (WebSocket, cron, etc.)
   190→    │       └── port 3003+ (each independent Bun project)
   191→    │
   192→    └── Database (SQLite dev / PostgreSQL prod)
   193→```
   194→
   195→### 3.3 لماذا هذه الاختيارات؟
   196→
   197→1. **SQLite للتطوير**: لا حاجة لخادم قاعدة بيانات، سريع، كافٍ لمعظم المشاريع. انتقل إلى PostgreSQL للإنتاج.
   198→2. **Clerk بدلاً من بناء auth من الصفر**: يوفّر أسابيع من العمل، يدعم متعدد المزودين، أمان جاهز.
   199→3. **shadcn/ui بدلاً من مكتبات UI الجاهزة**: الكود يصبح ملكك، قابل للتخصيص الكامل، لا قفل بائع.
   200→4. **Zustand بدلاً من Redux**: أبسط، أسرع، كافٍ لمعظم الحالات.
   201→5. **next-intl بدلاً من next-i18next**: مدمج أفضل مع App Router.
   202→6. **Bun بدلاً من npm/pnpm**: أسرع بـ 10-30 مرة في تثبيت الحزم، يدعم TypeScript أصلاً بدون `tsx`.

   206→## 4. القواعد الصارمة الـ15 (غير قابلة للتفاوض)
   207→
   208→> ⚠️ تطبيق هذه القواعد من البداية يمنع 90% من الأخطاء. تجاهل أي منها سيؤدي إلى أخطاء صعبة الإصلاح لاحقاً.
   209→
   210→### القاعدة 1: Prisma Singleton Pattern
   211→
   212→**المشكلة:** في التطوير، يحفظ Next.js الـ module، مما يُنشئ عدة اتصالات ويسبب `PrismaClientInitializationError` واستنفاد الاتصالات.
   213→
   214→```typescript
   215→// src/lib/db.ts
   216→import { PrismaClient } from '@prisma/client'
   217→
   218→const globalForPrisma = globalThis as unknown as {
   219→  prisma: PrismaClient | undefined
   220→}
   221→
   222→export const db =
   223→  globalForPrisma.prisma ??
   224→  new PrismaClient({
   225→    log: process.env.NODE_ENV === 'development' ? ['warn', 'error'] : ['error'],
   226→  })
   227→
   228→if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = db
   229→```
   230→
   231→**الاستخدام:** `import { db } from '@/lib/db'` — دائماً، في كل مكان.
   232→
   233→### القاعدة 2: مصادقة آمنة — `useSafeAuth()` و `useSafeUser()`
   234→
   235→**المشكلة:** `useUser()` من Clerk يُرجع `undefined` أثناء التحميل، مما يسبب crashes في الواجهة.
   236→
   237→```typescript
   238→// src/hooks/use-safe-auth.ts
   239→import { useAuth, useUser } from '@clerk/nextjs'
   240→
   241→export function useSafeAuth() {
   242→  const { isLoaded, isSignedIn, userId } = useAuth()
   243→  return {
   244→    isLoaded,
   245→    isSignedIn: isSignedIn ?? false,
   246→    userId: userId ?? null,
   247→  }
   248→}
   249→
   250→export function useSafeUser() {
   251→  const { user } = useUser()
   252→  return {
   253→    name: user?.firstName ?? '',
   254→    email: user?.emailAddresses?.[0]?.emailAddress ?? '',
   255→    imageUrl: user?.imageUrl ?? '',
   256→  }
   257→}
   258→```
   259→
   260→**في Server Components:**
   261→
   262→```typescript
   263→import { auth, currentUser } from '@clerk/nextjs/server'
   264→
   265→export async function getCurrentUser() {
   266→  const { userId } = await auth()
   267→  if (!userId) return null
   268→
   269→  const clerkUser = await currentUser()
   270→  if (!clerkUser) return null
   271→
   272→  // مزامنة مع قاعدة البيانات
   273→  return await db.user.findUnique({ where: { clerkId: userId } })
   274→}
   275→```

   277→### القاعدة 3: استجابات API موحّدة
   278→
   279→```typescript
   280→// src/lib/api-response.ts
   281→export interface PaginationMeta {
   282→  page: number
   283→  limit: number
   284→  total: number
   285→  totalPages: number
   286→}
   287→
   288→export function successResponse<T>(
   289→  data: T,
   290→  message: string,
   291→  meta?: PaginationMeta,
   292→  status = 200
   293→) {
   294→  return Response.json(
   295→    { success: true, data, message, ...(meta && { meta }) },
   296→    { status }
   297→  )
   298→}
   299→
   300→export function errorResponse(
   301→  code: string,
   302→  message: string,
   303→  field?: string,
   304→  status = 400
   305→) {
   306→  return Response.json(
   307→    { success: false, error: { code, message, field: field ?? null } },
   308→    { status }
   309→  )
   310→}
   311→```
   312→
   313→**التنسيق الموحّد:**
   314→
   315→```json
   316→// نجاح
   317→{ "success": true, "data": {...}, "message": "...", "meta": {...} }
   318→
   319→// خطأ
   320→{ "success": false, "error": { "code": "...", "message": "...", "field": null } }
   321→```
   322→
   323→### القاعدة 4: Decimal → `.toString()` دائماً
   324→
   325→عند إرجاع قيم Decimal من Prisma في JSON، يجب تحويلها إلى string دائماً لتجنب فقدان الدقة وأخطاء التسلسل.
   326→
   327→```typescript
   328→// ❌ خطأ: قد يفقد الدقة + يفشل JSON.stringify
   329→return successResponse({ price: event.price }, '...')
   330→
   331→// ✅ صحيح
   332→return successResponse({ price: event.price.toString() }, '...')
   333→```
   334→
   335→**لـ SQLite (الذي لا يدعم Decimal):** خزّنها كـ String بـ 3 منازل عشرية:
   336→
   337→```prisma
   338→price String  // "15.500" بدلاً من Decimal
   339→```
   340→
   341→**لـ PostgreSQL:** استخدم `Decimal(10, 3)` للعملة الكويتية (الدينار = 1000 فلس):
   342→
   343→```prisma
   344→price Decimal @db.Decimal(10, 3)
   345→```

   347→### القاعدة 5: أسماء الملفات kebab-case
   348→
   349→```
   350→✅ components/event-card.tsx
   351→✅ app/(dashboard)/events/page.tsx
   352→✅ lib/api-response.ts
   353→✅ hooks/use-safe-auth.ts
   354→
   355→❌ components/EventCard.tsx

   357→❌ components/event_card.tsx

   362→- المسارات (routes): `kebab-case` أيضاً: `/event-details/[slug]`

   364→### القاعدة 6: CSS Logical Properties (RTL-first)
   365→
   366→صمم وأنت تفكر في RTL أولاً، لأن العربية هي اللغة الأساسية. اختبر RTL قبل LTR.
   367→
   368→```css
   369→/* ❌ خطأ: لا يدعم RTL */
   370→margin-left: 16px;
   371→padding-right: 8px;
   372→text-align: left;
   373→.sidebar { left: 0; }
   374→.icon { margin-right: 8px; }
   375→
   376→/* ✅ صحيح: يدعم RTL تلقائياً */
   377→margin-inline-start: 16px;
   378→padding-inline-end: 8px;
   379→text-align: start;
   380→.sidebar { inset-inline-start: 0; }
   381→.icon { margin-inline-end: 8px; }
   382→```
   383→
   384→**في Tailwind 4:** استخدم `ms-`, `me-`, `ps-`, `pe-`, `start-0`, `end-0` بدلاً من `ml-`, `mr-`, `pl-`, `pr-`, `left-0`, `right-0`.

   386→### القاعدة 7: shadcn/ui فقط — لا مكونات من الصفر

   435→### القاعدة 10: catch blocks تستخدم `unknown` وليس `any`
   436→
   437→```typescript
   438→// ❌ خطأ
   439→try {
   440→  // ...
   441→} catch (error: any) {
   442→  console.log(error.message)
   443→  return errorResponse('ERR', error.message)
   444→}
   445→
   446→// ✅ صحيح
   447→try {
   448→  // ...
   449→} catch (error: unknown) {
   450→  const message = error instanceof Error ? error.message : 'حدث خطأ غير متوقع'
   451→  console.error('API error:', message, error)
   452→  return errorResponse('INTERNAL_ERROR', message, undefined, 500)
   453→}
   454→```

   456→### القاعدة 11: حماية الأدوار على مستوى الخادم
   457→
   458→تحقق من دور المستخدم في كل صفحة محمية وAPI endpoint، وليس فقط من تسجيل الدخول.
   459→
   460→```typescript
   461→// src/app/api/v1/admin/users/route.ts
   462→import { getCurrentUser } from '@/lib/auth-server'
   463→
   464→export async function GET() {
   465→  try {
   466→    const user = await getCurrentUser()
   467→    if (!user) {
   468→      return errorResponse('UNAUTHORIZED', 'يجب تسجيل الدخول', undefined, 401)
   469→    }
   470→    if (user.role !== 'ADMIN') {
   471→      return errorResponse('FORBIDDEN', 'غير مصرح', undefined, 403)
   472→    }
   473→    // ... المنطق
   474→  } catch (error: unknown) {
   475→    // ...
   476→  }
   477→}
   478→```
   479→
   480→**في middleware:**
   481→
   482→```typescript
   483→// src/middleware.ts
   484→import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server'
   485→
   486→const isAdminRoute = createRouteMatcher(['/admin(.*)'])
   487→const isOrganizerRoute = createRouteMatcher(['/organizer(.*)'])
   488→const isProtected = createRouteMatcher(['/dashboard(.*)', '/api/v1/(.*)'])
   489→
   490→export default clerkMiddleware(async (auth, req) => {
   491→  if (isAdminRoute(req)) {
   492→    await auth.protect({ role: 'org:admin' })
   493→  }
   494→  if (isOrganizerRoute(req)) {
   495→    await auth.protect({ role: 'org:member' })
   496→  }
   497→  if (isProtected(req)) {
   498→    await auth.protect()
   499→  }
   500→})
   501→```

   503→### القاعدة 12: لا `any` نهائياً
   504→
   505→**البدائل:**
   506→
   507→- `unknown` عندما لا تعرف النوع.
   508→- `Record<string, unknown>` للكائنات العامة.
   509→- تعريف `type` أو `interface` واضح.
   510→
   511→```typescript
   512→// ❌ خطأ
   513→function processData(data: any) {
   514→  return data.users
   515→}
   516→
   517→// ✅ صحيح
   518→interface UserData {
   519→  users: User[]
   520→}
   521→
   522→function processData(data: unknown): User[] {
   523→  if (typeof data !== 'object' || data === null) {
   524→    throw new Error('Invalid data')
   525→  }
   526→  if (!('users' in data)) {
   527→    throw new Error('Missing users field')
   528→  }
   529→  return (data as UserData).users
   530→}
   531→```

   534→
   535→كل منطق الخادم يكون في `src/app/api/v1/.../route.ts`. الواجهة تستدعيها عبر `fetch` أو `api-client.ts`.

   538→
   539→- API Routes أسهل في الاختبار.

   541→- فصل أوضح بين client و server.
   542→- توثيق API أوضح.

   691→### 6.1 المرحلة 0: التخطيط والتحضير (1-2 ساعة)
   692→
   693→**قبل كتابة أي سطر كود، يجب أن تكون لديك وثيقة متطلبات واضحة (PRD) تحتوي على:**
   694→
   695→- **الجمهور المستهدف**: من سيستخدم المنصة؟ ما احتياجاته؟
   696→- **الميزات الأساسية (MVP)**: ما الحد الأدنى من الميزات المطلوبة للإطلاق؟
   697→- **الميزات الثانوية**: ما الذي يمكن تأجيله للنسخ اللاحقة؟
   698→- **القيود التقنية**: ما اللغات، الأطر، والخدمات المطلوبة؟
   699→- **القيود التجارية**: ما الميزانية؟ ما الجدول الزمني؟
   700→
   701→**ثم:**
   702→
   703→1. أنشئ repository مع `.gitignore` صحيح.
   704→2. أعد branching strategy (`main`, `develop`, `feature/*`).
   705→3. جهّز بيئة التطوير (Bun, Docker, VS Code).
   706→4. أنشئ ملف `ARCHITECTURE.md` أولياً.

  1076→### 8.1 Prisma Schema — نموذج مرجعي
  1077→
  1078→```prisma
  1079→// prisma/schema.prisma
  1080→generator client {
  1081→  provider = "prisma-client-js"
  1082→}
  1083→
  1084→datasource db {
  1085→  provider = "postgresql"  // أو "sqlite" للتطوير المحلي
  1086→  url      = env("DATABASE_URL")
  1087→  directUrl = env("DIRECT_URL")  // مهم للـ migrations مع PostgreSQL
  1088→}
  1089→
  1090→// Enums (PostgreSQL فقط — SQLite لا يدعمها)
  1091→enum EventStatus {
  1092→  DRAFT
  1093→  PUBLISHED
  1094→  CANCELLED
  1095→  COMPLETED
  1096→}
  1097→
  1098→enum UserRole {
  1099→  USER
  1100→  ORGANIZER
  1101→  ADMIN
  1102→}
  1103→
  1104→enum Locale {
  1105→  ar
  1106→  en
  1107→}
  1108→
  1109→// النماذج
  1110→model User {
  1111→  id        String   @id @default(cuid())
  1112→  clerkId   String   @unique
  1113→  email     String   @unique
  1114→  name      String
  1115→  role      UserRole @default(USER)
  1116→  isActive  Boolean  @default(true)
  1117→
  1118→  // Relations
  1119→  events    Event[]
  1120→
  1121→  createdAt DateTime @default(now())
  1122→  updatedAt DateTime @updatedAt
  1123→
  1124→  @@index([email])
  1125→  @@map("users")
  1126→}
  1127→
  1128→model Event {
  1129→  id          String      @id @default(cuid())
  1130→  slug        String      @unique
  1131→  status      EventStatus @default(DRAFT)
  1132→  startDate   DateTime
  1133→  endDate     DateTime
  1134→  organizerId String
  1135→  organizer   User        @relation(fields: [organizerId], references: [id])
  1136→  categoryId  String
  1137→  category    Category    @relation(fields: [categoryId], references: [id])
  1138→
  1139→  // ترجمات JSONB (بديل أفضل من جدول منفصل)
  1140→  title       Json        // { "ar": "...", "en": "..." }
  1141→  description Json
  1142→
  1143→  // الحقول المالية — استخدم Decimal(10, 3) للعملة الكويتية
  1144→  minPrice    Decimal?    @db.Decimal(10, 3)
  1145→
  1146→  ticketTiers TicketTier[]
  1147→  createdAt   DateTime    @default(now())
  1148→  updatedAt   DateTime    @updatedAt
  1149→
  1150→  @@index([status, startDate])
  1151→  @@index([categoryId])
  1152→  @@index([organizerId])
  1153→  @@map("events")
  1154→}
  1155→
  1156→model TicketTier {
  1157→  id        String  @id @default(cuid())
  1158→  eventId   String
  1159→  event     Event   @relation(fields: [eventId], references: [id], onDelete: Cascade)
  1160→  name      Json    // { "ar": "...", "en": "..." }
  1161→  price     Decimal @db.Decimal(10, 3)
  1162→  quantity  Int
  1163→  sold      Int     @default(0)
  1164→
  1165→  @@index([eventId])
  1166→  @@map("ticket_tiers")
  1167→}
  1168→
  1169→model Category {
  1170→  id    String  @id @default(cuid())
  1171→  slug  String  @unique
  1172→  name  Json    // { "ar": "...", "en": "..." }
  1173→  events Event[]
  1174→
  1175→  @@map("categories")
  1176→}
  1177→```
  1178→
  1179→**دروس مهمة من الـ schema:**
  1180→
  1181→- استخدم `Decimal(10, 3)` للعملة الكويتية (الدينار = 1000 فلس)، وليس `Decimal(10, 2)`.
  1182→- استخدم `Json` لحقل الترجمات متعددة اللغات بدلاً من جدول منفصل.
  1183→- أضف `@@index` على الحقول المستخدمة بكثرة في `WHERE` و `ORDER BY`.
  1184→- استخدم `onDelete: Cascade` بحذر — فقط عندما يكون منطقياً (مثل حذف تذاكر الفعالية عند حذف الفعالية).
  1185→- استخدم `@map("snake_case")` لأسماء الأعمدة، `@@map("plural_table")` لأسماء الجداول.
  1186→- استخدم `cuid()` للـ IDs (أفضل من UUID للـ URLs).
  1187→- **SQLite لا يدعم `enum` و `Decimal`** — خزّنها كـ `String`.

  1214→### 8.3 API Route — قالب قياسي
  1215→
  1216→```typescript
  1217→// src/app/api/v1/events/route.ts
  1218→import { NextRequest } from 'next/server'
  1219→import { db } from '@/lib/db'
  1220→import { successResponse, errorResponse } from '@/lib/api-response'
  1221→import { getCurrentUser } from '@/lib/auth-server'
  1222→import { eventCreateSchema } from '@/lib/validators/event-schema'
  1223→import { z } from 'zod'
  1224→
  1225→const querySchema = z.object({
  1226→  page: z.coerce.number().min(1).default(1),
  1227→  limit: z.coerce.number().min(1).max(50).default(10),
  1228→  locale: z.enum(['ar', 'en']).default('ar'),
  1229→  status: z.enum(['DRAFT', 'PUBLISHED', 'CANCELLED', 'COMPLETED']).optional(),
  1230→})
  1231→
  1232→export async function GET(req: NextRequest) {
  1233→  try {
  1234→    // 1. المصادقة
  1235→    const currentUser = await getCurrentUser()
  1236→    if (!currentUser) {
  1237→      return errorResponse('UNAUTHORIZED', 'يجب تسجيل الدخول', undefined, 401)
  1238→    }
  1239→
  1240→    // 2. التحقق من المدخلات
  1241→    const { searchParams } = new URL(req.url)
  1242→    const { page, limit, locale, status } = querySchema.parse(
  1243→      Object.fromEntries(searchParams)
  1244→    )
  1245→
  1246→    // 3. الاستعلام
  1247→    const where = {
  1248→      ...(status && { status }),
  1249→    }
  1250→
  1251→    const [events, total] = await Promise.all([
  1252→      db.event.findMany({
  1253→        where,
  1254→        skip: (page - 1) * limit,
  1255→        take: limit,
  1256→        orderBy: { startDate: 'asc' },
  1257→        include: {
  1258→          category: true,
  1259→          ticketTiers: { select: { id: true, price: true, quantity: true, sold: true } },
  1260→        },
  1261→      }),
  1262→      db.event.count({ where }),
  1263→    ])
  1264→
  1265→    // 4. تحويل Decimal إلى String (القاعدة 4)
  1266→    const safeEvents = events.map((event) => ({
  1267→      ...event,
  1268→      minPrice: event.minPrice?.toString() ?? null,
  1269→      ticketTiers: event.ticketTiers.map((tier) => ({
  1270→        ...tier,
  1271→        price: tier.price.toString(),
  1272→      })),
  1273→    }))
  1274→
  1275→    // 5. الإرجاع
  1276→    return successResponse(safeEvents, 'تم جلب الفعاليات', {
  1277→      page,
  1278→      limit,
  1279→      total,
  1280→      totalPages: Math.ceil(total / limit),
  1281→    })
  1282→  } catch (error: unknown) {
  1283→    if (error instanceof z.ZodError) {
  1284→      return errorResponse(
  1285→        'VALIDATION_ERROR',
  1286→        'بارامترات غير صالحة',
  1287→        undefined,
  1288→        422
  1289→      )
  1290→    }
  1291→    const message = error instanceof Error ? error.message : 'خطأ غير متوقع'
  1292→    console.error('GET /api/v1/events:', message, error)
  1293→    return errorResponse('INTERNAL_ERROR', message, undefined, 500)
  1294→  }
  1295→}
  1296→
  1297→export async function POST(req: NextRequest) {
  1298→  try {
  1299→    // 1. المصادقة + الصلاحيات
  1300→    const currentUser = await getCurrentUser()
  1301→    if (!currentUser) {
  1302→      return errorResponse('UNAUTHORIZED', 'يجب تسجيل الدخول', undefined, 401)
  1303→    }
  1304→    if (currentUser.role !== 'ORGANIZER' && currentUser.role !== 'ADMIN') {
  1305→      return errorResponse('FORBIDDEN', 'غير مصرح', undefined, 403)
  1306→    }
  1307→
  1308→    // 2. التحقق من المدخلات
  1309→    const body = await req.json()
  1310→    const validated = eventCreateSchema.parse(body)
  1311→
  1312→    // 3. الإنشاء
  1313→    const event = await db.event.create({
  1314→      data: {
  1315→        ...validated,
  1316→        organizerId: currentUser.id,
  1317→        slug: validated.slug || generateSlug(validated.title),
  1318→      },
  1319→    })
  1320→
  1321→    return successResponse(event, 'تم إنشاء الفعالية', undefined, 201)
  1322→  } catch (error: unknown) {
  1323→    if (error instanceof z.ZodError) {
  1324→      return errorResponse(
  1325→        'VALIDATION_ERROR',
  1326→        'بيانات غير صالحة',
  1327→        error.errors[0]?.path[0]?.toString(),
  1328→        422
  1329→      )
  1330→    }
  1331→    const message = error instanceof Error ? error.message : 'خطأ غير متوقع'
  1332→    console.error('POST /api/v1/events:', message, error)
  1333→    return errorResponse('INTERNAL_ERROR', message, undefined, 500)
  1334→  }
  1335→}
  1336→
  1337→function generateSlug(title: { ar: string; en: string }): string {
  1338→  return title.en
  1339→    .toLowerCase()
  1340→    .replace(/[^a-z0-9]+/g, '-')
  1341→    .replace(/^-+|-+$/g, '')
  1342→}

  1346→
  1347→```typescript
  1348→// src/lib/validators/event-schema.ts
  1349→import { z } from 'zod'
  1350→
  1351→export const eventCreateSchema = z.object({
  1352→  title: z.object({
  1353→    ar: z.string().min(3, { message: 'العنوان بالعربية قصير جداً' }),
  1354→    en: z.string().min(3, { message: 'العنوان بالإنجليزية قصير جداً' }),
  1355→  }),
  1356→  description: z.object({
  1357→    ar: z.string().max(2000, { message: 'الوصف طويل جداً' }),
  1358→    en: z.string().max(2000, { message: 'الوصف طويل جداً' }),
  1359→  }),
  1360→  startDate: z.string().datetime({ message: 'تاريخ البداية غير صالح' }),
  1361→  endDate: z.string().datetime({ message: 'تاريخ النهاية غير صالح' }),
  1362→  categoryId: z.string().cuid({ message: 'معرّف الفئة غير صالح' }),
  1363→  slug: z
  1364→    .string()
  1365→    .min(3)
  1366→    .max(100)
  1367→    .regex(/^[a-z0-9-]+$/, { message: 'الـ slug يجب أن يكون أحرف صغيرة وأرقام وشرطات' })
  1368→    .optional(),
  1369→  ticketTiers: z
  1370→    .array(
  1371→      z.object({
  1372→        name: z.object({
  1373→          ar: z.string().min(2),
  1374→          en: z.string().min(2),
  1375→        }),
  1376→        price: z
  1377→          .string()
  1378→          .refine((val) => {
  1379→            const num = parseFloat(val)
  1380→            return !isNaN(num) && num >= 0 && num < 1000000
  1381→          }, { message: 'السعر غير صالح' }),
  1382→        quantity: z.number().int().min(1).max(100000),
  1383→      })
  1384→    )
  1385→    .min(1, { message: 'يجب إضافة فئة تذاكر واحدة على الأقل' }),
  1386→})
  1387→
  1388→export type EventCreateInput = z.infer<typeof eventCreateSchema>

  1397→import { redirect } from 'next/navigation'
  1398→
  1399→export interface SafeUser {
  1400→  id: string
  1401→  clerkId: string
  1402→  email: string
  1403→  name: string
  1404→  role: 'USER' | 'ORGANIZER' | 'ADMIN'
  1405→  isActive: boolean
  1406→}
  1407→
  1408→export async function getCurrentUser(): Promise<SafeUser | null> {
  1409→  try {
  1410→    const { userId } = await auth()
  1411→    if (!userId) return null
  1412→
  1413→    const clerkUser = await currentUser()
  1414→    if (!clerkUser) return null
  1415→
  1416→    // مزامنة مع قاعدة البيانات
  1417→    const dbUser = await db.user.findUnique({
  1418→      where: { clerkId: userId },
  1419→    })
  1420→
  1421→    if (!dbUser) {
  1422→      // أنشئ المستخدم تلقائياً إذا لم يكن موجوداً (auto-provisioning)
  1423→      const newUser = await db.user.create({
  1424→        data: {
  1425→          clerkId: userId,
  1426→          email: clerkUser.emailAddresses[0]?.emailAddress ?? '',
  1427→          name: `${clerkUser.firstName ?? ''} ${clerkUser.lastName ?? ''}`.trim(),
  1428→          role: 'USER',
  1429→        },
  1430→      })
  1431→      return newUser
  1432→    }
  1433→
  1434→    return dbUser
  1435→  } catch (error) {
  1436→    console.error('getCurrentUser error:', error)
  1437→    return null
  1438→  }
  1439→}
  1440→
  1441→export async function requireUser(): Promise<SafeUser> {
  1442→  const user = await getCurrentUser()
  1443→  if (!user) {
  1444→    redirect('/login')
  1445→  }
  1446→  return user
  1447→}
  1448→
  1449→export async function requireRole(...roles: SafeUser['role'][]): Promise<SafeUser> {
  1450→  const user = await requireUser()
  1451→  if (!roles.includes(user.role)) {
  1452→    redirect('/unauthorized')
  1453→  }
  1454→  return user
  1455→}
  1456→```

  1598→
  1599→```typescript
  1600→// src/lib/api-client.ts
  1601→interface ApiResponse<T> {
  1602→  success: boolean
  1603→  data?: T
  1604→  message?: string
  1605→  error?: { code: string; message: string; field?: string | null }
  1606→  meta?: { page: number; limit: number; total: number; totalPages: number }
  1607→}
  1608→
  1609→class ApiClient {
  1610→  private baseUrl: string
  1611→
  1612→  constructor() {
  1613→    this.baseUrl = process.env.NEXT_PUBLIC_APP_URL ?? ''
  1614→  }
  1615→
  1616→  private async request<T>(
  1617→    path: string,
  1618→    options: RequestInit = {}
  1619→  ): Promise<ApiResponse<T>> {
  1620→    try {
  1621→      const response = await fetch(`${this.baseUrl}/api/v1${path}`, {
  1622→        ...options,
  1623→        headers: {
  1624→          'Content-Type': 'application/json',
  1625→          ...options.headers,
  1626→        },
  1627→        // تأكد من إرسال cookies (للمصادقة)
  1628→        credentials: 'include',
  1629→      })
  1630→
  1631→      const data = await response.json()
  1632→
  1633→      if (!response.ok || !data.success) {
  1634→        throw new Error(data.error?.message ?? 'Request failed')
  1635→      }
  1636→
  1637→      return data
  1638→    } catch (error) {
  1639→      console.error(`API ${path} error:`, error)
  1640→      throw error
  1641→    }
  1642→  }
  1643→
  1644→  get<T>(path: string) {
  1645→    return this.request<T>(path)
  1646→  }
  1647→
  1648→  post<T>(path: string, body: unknown) {
  1649→    return this.request<T>(path, {
  1650→      method: 'POST',
  1651→      body: JSON.stringify(body),
  1652→    })
  1653→  }
  1654→
  1655→  patch<T>(path: string, body: unknown) {
  1656→    return this.request<T>(path, {
  1657→      method: 'PATCH',
  1658→      body: JSON.stringify(body),
  1659→    })
  1660→  }
  1661→
  1662→  delete<T>(path: string) {
  1663→    return this.request<T>(path, { method: 'DELETE' })
  1664→  }
  1665→}
  1666→
  1667→export const apiClient = new ApiClient()
  1668→```

  1979→### 9.2 Payment Integration — مبادئ آمنة
  1980→
  1981→1. **لا تثق بالواجهة أبداً** — كل التحقق في الخادم.
  1982→2. **Idempotency** — استخدم `bookingId` فريد لمنع الدفع المزدوج.
  1983→3. **Webhook للتحقق** — لا تعتمد على redirect العودة من المستخدم.
  1984→4. **Transaction** — استخدم Prisma `$transaction` لضمان الاتساق.
  1985→
  1986→```typescript
  1987→// src/app/api/v1/payments/create/route.ts
  1988→import { NextRequest } from 'next/server'
  1989→import { db } from '@/lib/db'
  1990→import { successResponse, errorResponse } from '@/lib/api-response'
  1991→import { getCurrentUser } from '@/lib/auth-server'
  1992→import { Decimal } from '@prisma/client/runtime/library'
  1993→
